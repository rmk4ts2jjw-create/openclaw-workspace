## Ghost Dispatch Fix — 2026-06-12 23:47 BST

**Problem:** Tasks moved to 'In Progress' were reverting to 'Backlog' after 5-10 minutes, then getting re-dispatched in an infinite loop. Root causes:

1. **No pre-dispatch validation** — dispatcher set status to in_progress + 'Agent starting…' without checking if the task was actually dispatchable
2. **No ghost timeout** — if no sub-agent picked up the task, it sat in 'Agent starting…' for 30 min until stall detector fired
3. **wasStalled permanent exclusion bug** — Phase 2 cleared stalledAt but left wasStalled=True, permanently preventing re-dispatch
4. **dispatch→reset→dispatch loop** — after stall reset, tasks were immediately re-dispatched

**Fixes applied:**
1. Pre-dispatch validation: check excluded tags BEFORE setting in_progress. If failed, set dispatchFailed=True with reason
2. 60s ghost timeout (Phase 0): if currentStep is 'Agent starting…' and lastActivity >60s, reset immediately
3. Phase 2 now clears wasStalled when clearing stalledAt
4. Phase 0 ghost reset sets wasStalled=True
5. MC UI: 'Dispatch Failed' red badge + 'Ghost' orange badge on TaskCard
6. Task type: added dispatchFailed, dispatchFailedReason, dispatchFailedAt, ghostDispatch fields
7. TaskHistoryEntry: added stalled_reset, ghost_dispatch_reset, dispatch_failed, stalled_cleared actions
8. HEARTBEAT.md: updated dispatch rules — never set currentStep to 'Agent starting…', validate before dispatch
