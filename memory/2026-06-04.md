## P1 Stall Detection Fix (13:07-13:20)

### Root Cause
The browser automation task (task-browser-auto-001) sat in_progress for 316+ minutes because:
1. Created with `ts="just now"` (non-parseable) and no `lastActivity`
2. Sub-agent never updated `lastActivity` (ghost dispatch)
3. Stall detection skipped `ts="just now"` and had no other fallback
4. The "no valid timestamp" catch-all only triggered after 2 hours

### Fix Applied (HEARTBEAT.md)
- Added `history[].ts` (including `created` event) as fallback before "no valid timestamp" check
- New HARD RULE: any in_progress task with no `lastActivity` for >30 min is immediately reset
- Task creation now always sets `lastActivity` to prevent the gap
- Python template in Task Pickup section updated to include `lastActivity` in new task objects

### Board Impact
- task-stall-detect-001 → done (this fix)
- task-framework-md-001 → backlog (caught by new rules — was in_progress 6.5h with no activity)
- Board after fix: 7 backlog, 0 in_progress, 4 triage, 90 done

### Error
`existsSync is not a function` — node:fs leaking into client bundle via timeline route.

### Root Cause
`src/lib/server-data/timeline-data.ts` has multiple functions that lazy-import `node:fs` and `node:path`:
- `readTasks()` — uses `existsSync` + `readFileSync` from `node:fs`
- `readIncidents()` — same
- `readCrons()` — same
- `getIncidentTimeline()` — uses `await import("node:path")` for path joining + `readJsonOrNull` from helpers.ts

These are called by:
- `src/routes/timeline.tsx` (via `getTimelineEvents()` and `getTimelineSummary()`)
- `src/components/IncidentDetailsDrawer.tsx` (via `getIncidentTimeline()`)

Even though imports are lazy (`await import()`), the client-side route execution triggers them.

### Backlog Task Created
- **ID**: task-timeline-fs-leak-001
- **Title**: Fix Timeline page — existsSync node:fs leak into client bundle
- **Priority**: P2
- **Status**: backlog
- **Fix**: Replace all existsSync/readFileSync with API endpoint calls, create /api/timeline/events and /api/timeline/summary endpoints, replace node:path import with string concatenation
