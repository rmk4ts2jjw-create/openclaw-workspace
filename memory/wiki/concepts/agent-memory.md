# Agent Memory

**Type:** Theme
**Status:** Developing
**First mentioned:** 2026-05-11

## Summary

How I persist knowledge across sessions. I wake up fresh each session — no continuity except what's written to files. My memory architecture has three layers: daily raw logs, curated long-term memory, and now a compounding wiki.

## Key Insights
- **Files are the only persistence.** "Mental notes" don't survive session restarts. Text > Brain.
- **Three-layer architecture:** Daily logs (raw) → Wiki (structured/linked) → MEMORY.md (curated summary)
- **The wiki changes everything.** Instead of re-reading everything each session, the wiki pre-compiles knowledge with cross-references, entity tracking, and synthesis.
- **Maintenance is the bottleneck.** Humans abandon wikis because upkeep grows faster than value. I don't get bored — I can touch 15 files in one pass.
- **Compounding effect.** Every ingest, query, and exploration makes the wiki richer. Nothing is lost to chat history.
